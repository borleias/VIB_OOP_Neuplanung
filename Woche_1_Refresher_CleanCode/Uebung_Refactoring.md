# Übung: Das "Spaghetti-Problem"

## Aufgabe
Gegeben ist ein Programm, das die Verwaltung von Bibliotheksnutzern simuliert. Das Programm funktioniert zwar, ist aber extrem schwer zu lesen, zu warten und zu testen.

**Deine Mission:** Refaktoriere den Code unter Anwendung der Clean Code Prinzipien aus dem theoretischen Teil.

### Schritte:
1.  **Lies den Code:** Versuche zu verstehen, was passiert.
2.  **Identifiziere Code Smells:** Wo sind unklare Namen? Wo sind zu lange Methoden? Wo sind unnötige Kommentare?
3.  **Wende Refactoring-Techniken an:**
    -   Extrahiere Methoden (Extract Method).
    -   Benenne Variablen und Methoden um (Rename).
    -   Verwende sprechende Klassen.
4.  **Prüfe das Ergebnis:** Funktioniert das Programm nach deinem Refactoring noch genau so wie vorher?

### Ziel:
Ein modularer, lesbarer und wartbarer Code, der die Absicht des Autors klar kommuniziert.
