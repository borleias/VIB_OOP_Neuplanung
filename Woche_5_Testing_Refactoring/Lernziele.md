# Lernziele Woche 5: Qualitätssicherung & Refactoring

Die Studierenden sollen am Ende der fünften Woche:
1.  **Unit Testing beherrschen:** Schreiben von Tests für isolierte Klassen (xUnit oder NUnit).
2.  **Testbare Software entwerfen:** Wissen, warum Interfaces für Mocking so wichtig sind.
3.  **TDD Basics (Test-Driven Development) kennen:** Wissen, wie man erst den Test und dann den Code schreibt (Red-Green-Refactor).
4.  **Refactoring-Katalog nutzen:** Gezielte Techniken kennen, um Code "schön" zu halten.

1.  Vortrag Testing (45 Min).
2.  Live-Coding Unit Tests (45 Min).
