# Lernziele Woche 4: SOLID-Prinzipien & Architektur

Die Studierenden sollen am Ende der vierten Woche:
1.  **SOLID-Prinzipien im Detail verstehen:** Single Responsibility, Open/Closed, Liskov Substitution, Interface Segregation, Dependency Inversion.
2.  **Architektur-Grundlagen kennen:** Was ist Schichtenarchitektur (3-Tier)? Warum ist Trennung von UI und Business-Logik essenziell?
3.  **Dependency Injection (DI) einsetzen:** Wissen, wie man Abhängigkeiten von außen in Klassen "hineingibt" (statt sie intern mit `new` zu erzeugen).
4.  **Die Vorteile loser Kopplung erkennen:** Code besser testbar und erweiterbar machen.

---

## Ablauf
1.  Vortrag SOLID (90 Min): Theorie am Beispiel schlechten Codes.
2.  Einführung Schichtenarchitektur (30 Min).
3.  Workshop (60 Min): Refactoring einer kleinen App auf SOLID.
