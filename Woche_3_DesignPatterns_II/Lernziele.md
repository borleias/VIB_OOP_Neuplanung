# Lernziele Woche 3: Design Patterns II (Verhalten)

Die Studierenden sollen am Ende der dritten Woche:
1.  **Das Strategy Pattern anwenden:** Algorithmen austauschbar machen (z.B. verschiedene Rabatt-Logiken).
2.  **Das Observer Pattern verstehen:** Wissen, wie Objekte über Zustandsänderungen informiert werden, ohne eng gekoppelt zu sein.
3.  **Das State Pattern einsetzen:** Komplexes Verhalten, das vom internen Zustand abhängt, sauber strukturieren (statt Riesen-Switch-Case).
4.  **Die Unterschiede der Verhaltensmuster kennen:** Wissen, welches Muster für welches Kommunikationsproblem am besten geeignet ist.

---

## Ablauf
1.  Wiederholung Woche 2 (15 Min).
2.  Theorie Strategy Pattern (45 Min): Beispiel Sortier-Algorithmen oder Preisberechnung.
3.  Theorie Observer Pattern (45 Min): C# Events vs. "klassisches" Observer Pattern.
4.  Theorie State Pattern (30 Min): Zustandsdiagramm zu Code.
5.  Praktische Übung (45 Min): Implementierung eines Workflow-Systems mit Strategy.
